# 05 · Social — Weekly Report (weekly)

ROUTINE STANDARD:
- TRIGGER: Sunday 20:00 {{TIMEZONE}}.
- STRATEGIC GOAL: turn the week's short-form performance into concrete guidance — what the founder should shoot next, and when to post.
- SCOPE: read-only analytics + learnings + framework-table maintenance. NOT in scope: creating/editing posts, media, or store data.
- AUTONOMY: fully autonomous. Sanctioned writes: social-learnings.md append, post-outcomes.jsonl append, monthly framework-table refresh, the report page.
- KPIs: views, watch/completion, follower growth, profile clicks, referral sessions — per platform, WoW.
- LOGGING & HONESTY: new channels have thin data for WEEKS — "no data yet" is the honest report; never pad or fabricate.

STEPS:
1. WINDOW: completed Mon–Sun vs prior 7 days. From ledger.json: posts per platform this week.
2. ANALYTICS via {{SCHEDULER}} per network in {{SOCIAL_NETWORKS}}: views, engagement, watch/completion where exposed, followers, profile clicks. Top 3 / bottom 3 posts with one-line diagnoses.
3. FEATURE×OUTCOME JOIN: join each post's metrics with its ledger `features` → APPEND snapshot-dated rows to post-outcomes.jsonl (append-only — the compounding dataset). Grade against the 2026 algorithm gates: TikTok ≥70% completion = viral-grade; Shorts = watch-time-per-impression, 30–45s sweet spot (~65% AVD gate <30s, ~50% over); Reels = watch %, likes/reach, sends/reach; saves ≈2–3× likes; the first 3 seconds drive ~80% of completion variance; ~85% watched on mute. n<10 per cell = "data too thin".
4. REFERRALS: store sessions from tiktok/instagram/youtube utm or referrers, WoW. Zero is expected early.
5. WHAT TO SHOOT NEXT: exactly 3 concrete, evidence-based suggestions (format + aesthetic + subject + why — every one built around a first-3-seconds hook and mute-proof design). When own data is thin, borrow proven demand signals from the Pinterest Weekly Audit's trending section and label the source.
6. MONTHLY (first Sunday): refresh the framework's time table (own scheduler best-time data > generic research) and hashtag banks; note changes in the framework changelog.
7. LEARNINGS: append a dated section to social-learnings.md (latest section stands alone): FORMAT LEARNINGS / SLOT LEARNINGS / HASHTAG LEARNINGS / SHOOT LIST / EXPERIMENT (once ≥2 weeks of data: verdict last week's first, then at most ONE mechanical caption/slot experiment for the daily routine, or NONE) / NOTES. Every line cites numbers.
8. OUTPUT: report page (Notion "Social Weekly Report" parent) + chat TL;DR: 3 lines + the 3 shoot suggestions.
